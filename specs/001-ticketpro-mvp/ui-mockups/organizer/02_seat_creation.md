## 画面要素

### 一括登録フォーム

| 項目 | 必須 | 形式 | バリデーション |
|-----|------|------|--------------|
| ブロック名 | ✓ | テキスト | 50文字以内 |
| 座席種別 | ✓ | セレクトボックス | S席/A席/B席 |
| 列の範囲 | ✓ | テキスト | 英字(A-Z)または数字(1-99) |
| 番号の範囲 | ✓ | 数値 | 1-999 |

### 座席一覧テーブル

| カラム | 説明 |
|-------|------|
| ブロック | ブロック名 |
| 列 | 列番号 |
| 番号 | 座席番号 |
| 座席種別 | S席/A席/B席 |
| ステータス | 空席/予約中/売約済 |
| アクション | [削除]ボタン |

### フィルター機能

- ブロック選択（セレクトボックス）
- 座席種別選択（セレクトボックス）
- [検索]ボタンでフィルタリング

## 動作フロー

### 正常フロー

1. 主催者が会場一覧から会場を選択 → 「座席表作成」をクリック
2. ブロック名、座席種別、列・番号の範囲を入力
3. プレビューで生成される座席を確認
4. 「座席を一括登録」ボタンをクリック
5. 座席がデータベースに保存される
6. 成功メッセージ「200席を登録しました」
7. 登録済み座席一覧に反映

### エラーフロー

1. 必須項目未入力
   - エラーメッセージ「必須項目を入力してください」

2. 重複座席の登録
   - エラーメッセージ「既に登録されている座席があります: 1階席-A-1」

3. 列の範囲が不正
   - エラーメッセージ「列の範囲はA-Zまたは1-99の範囲で指定してください」

## 座席一括登録の例

### 入力例1: 1階席（S席）
```
ブロック名: 1階席
座席種別: S席
列の範囲: A ～ J
番号の範囲: 1 ～ 20
→ 生成: 1階席-A-1, 1階席-A-2, ..., 1階席-J-20 (200席)
```

### 入力例2: 2階席（A席）
```
ブロック名: 2階席
座席種別: A席
列の範囲: 1 ～ 10
番号の範囲: 1 ～ 20
→ 生成: 2階席-1-1, 2階席-1-2, ..., 2階席-10-20 (200席)
```

## レスポンシブ対応

### PC (1024px以上)
- 登録フォームと一覧を横並び表示

### タブレット (768px - 1023px)
- 登録フォームと一覧を縦並び表示
- テーブルはスクロール可能

### スマホ (767px以下)
- 登録フォームはフルサイズ
- テーブルはカード形式で表示
