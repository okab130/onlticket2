## 画面要素

### 入力項目

| 項目 | 必須 | 形式 | バリデーション |
|-----|------|------|--------------|
| イベント名 | ✓ | テキスト | 200文字以内 |
| カテゴリ | ✓ | セレクトボックス | コンサート、スポーツ、演劇、フェス、オンライン |
| イベント説明 | ✓ | テキストエリア | 2000文字以内 |
| イベント画像 | - | ファイルアップロード | JPG/PNG、5MB以内 |
| 会場 | ✓ | セレクトボックス | 既存会場から選択 |
| 開始日時 | ✓ | 日付+時刻 | 現在日時より未来 |
| 終了日時 | - | 日付+時刻 | 開始日時より後 |

### チケット種別設定

| 項目 | 必須 | 形式 | バリデーション |
|-----|------|------|--------------|
| 種別名 | ✓ | テキスト | 100文字以内 |
| タイプ | ✓ | ラジオボタン | 座席指定/自由席 |
| 価格 | ✓ | 数値 | 0以上 |
| 販売枚数 | ✓ | 数値 | 1以上 |
| 販売期間 | - | 日付範囲 | 販売開始≦イベント開始日時 |

### ボタン

- **[キャンセル]**: イベント一覧へ戻る（入力内容は破棄）
- **[下書き保存]**: 下書きとして保存（is_public=False）
- **[イベントを登録]**: 入力内容を検証して登録、公開設定に従って公開

---

## 動作フロー

### 正常フロー

1. 主催者が「イベント管理」→「新規イベント作成」をクリック
2. フォームに必要情報を入力
3. チケット種別を1つ以上追加
4. 「イベントを登録」ボタンをクリック
5. バリデーション成功 → イベントが登録される
6. 成功メッセージ表示「イベントを登録しました」
7. イベント詳細画面へリダイレクト

### エラーフロー

1. 必須項目未入力の場合
   - エラーメッセージ表示「必須項目を入力してください」
   - 該当項目を赤枠でハイライト

2. バリデーションエラー
   - 各項目の下にエラーメッセージ表示
   - 例: 「開始日時は現在日時より未来の日時を指定してください」

3. 会場未選択
   - エラーメッセージ「会場を選択してください」
   - [新規会場登録]リンクを表示

## レスポンシブ対応

### PC (1024px以上)
- 2カラムレイアウト（左: 基本情報、右: プレビュー）
- チケット種別は横並び

### タブレット (768px - 1023px)
- 1カラムレイアウト
- チケット種別は縦並び

### スマホ (767px以下)
- 1カラムレイアウト
- 日付と時刻は縦並び
- ボタンはフルサイズ

## Constitution遵守チェック

✅ **データモデル中心**: Event, TicketType, Venueモデルと連携  
✅ **入力項目の事前参照**: 会場選択は既存会場から選択（会場マスタ参照）  
✅ **バリデーション**: 必須項目、日時の論理チェック  
✅ **エラーハンドリング**: 明確なエラーメッセージ表示  
✅ **レスポンシブデザイン**: Bootstrap 5で実装
